# Sprint 8 — Scenario Builder + Synthdata Factory

**Duration:** Weeks 15-16, **only after pre-S8 gate**
**Persona promise:** An admin can describe a new synthetic pilot, review a cited plan, build a safe bundle, run it in an isolated preview, and deploy it to G0 with provenance and guardrails.
**Depends on:** Sprint 7 (motor-fleet shipped, audit bundle accepted by compliance, synthetic-data factory pattern proven). **Pre-S8 gate must pass.**

---

## Pre-S8 Gate (run BEFORE Sprint 8)

Before Sprint 8 starts, run an isolated experiment to test the full Plan → Build → Lint → Preview pipeline. **Five diverse briefs**, end-to-end, generated by the prototype Builder.

**Pass criteria:**
- 4 of 5 produce a valid plan structure.
- 4 of 5 pass bundle lint.
- 4 of 5 preview runs complete.
- **0** generated artefacts violate security policy.
- Average session cost under the agreed cap (e.g., $5 per session).

**If the gate fails:** S8 becomes "Builder Experiment Hardening" — improve the pipeline, expand the test set, do not ship the product surface yet.

---

## Why This Sprint Exists

Sprints 1–7 build a platform with two pilots. Sprint 8 turns it into a **pilot factory**: an admin describes a domain in plain language, the Builder produces a researched plan, generates synthetic data, scaffolds the flow, lints it for security, runs a preview in a sandboxed environment, and deploys it to G0 — with full provenance and a hard human gate before G0→G1.

This is the highest-risk sprint because the Builder generates artefacts that the gateway will execute. Every guardrail (capability manifest, deny-by-default egress, policy checks, isolated preview, signed provenance) exists to prevent the Builder from generating a workflow that exfiltrates secrets, calls disallowed external endpoints, or impersonates a real claim.

## Scope Summary

### In Scope

**Builder backend (`gateway/src/gateway/builder/`):**
- `nim.py` — NVIDIA NIM (or equivalent local-friendly LLM) client wrapper for cost control. Token budget per session enforced.
- `prompts.py` — Builder prompts as Langfuse Prompts (versioned).
- `session.py` — plain-Python deterministic FSM: `intake → research → plan → approve → build → lint → preview → deploy`. Advisory PG lock per session.
- `recipes.py` — building blocks (Property recipe, Motor recipe, generic Triage recipe). The Builder selects/combines.
- `api.py` — FastAPI routes for the `/builder` UI: `POST /builder/sessions`, `POST /builder/sessions/{id}/turn`, `POST /builder/sessions/{id}/approve`, etc.
- `tools/web_search_insurance.py` — citation tool restricted to allowlisted insurance/regulator domains. Returns `(text, url, snippet, retrieved_at)`.
- `tools/bundle_lint.py` — OPA/Rego (or simpler equivalent) policy checks on generated bundle:
  - Capability manifest required.
  - Egress targets in allowlist.
  - No env-var access from generated n8n Code nodes.
  - No literal secrets in artefacts.
- `tools/preview_run.py` — runs the bundle inside an isolated Railway project (no production credentials, separate Supabase test schema, separate Langfuse project).

**Builder frontend (`/builder`):**
- Chat pane (intake + plan iteration).
- Plan preview panel (sectioned: Persona, Data, Flow, Eval, KPIs, Citations).
- Monaco diff for build artefacts (flow JSON, eval dataset, prompts).
- Flow graph preview (read-only Langflow canvas).
- Dataset manifest preview.
- Preview run trace deep-link.
- **Deploy G0** button (writes to G0 only, no G0→G1 promotion here).
- Cancel/Stop generation button.
- Token/cost budget banner with hard stop (no overrun without re-confirmation).

**Synthdata factory (`gateway/synthdata/`):**
- Seed upload: operator can upload a small CSV; gateway fingerprints and forbids same fingerprint reuse to prevent re-leak.
- Dataset manifest schema (provenance, generators used, validators passed, count).
- Faker/Pydantic-driven generation (extended motor-fleet pattern).
- Document/email rendering (Jinja templates, multi-language scaffolds).
- Validators:
  - **Schema** — every row matches Pydantic.
  - **Distribution** — categoricals/numerics within target stats.
  - **Anti-PII** — regex + entropy checks; reject if any string matches `EMAIL/IBAN/PHONE`.
  - **Persona fidelity** — claims content matches declared persona profile (LLM check).

**Security:**
- Generated workflow capability manifest — declares every external call site; lint rejects undeclared calls.
- Deny-by-default egress allowlist; n8n Function/Code nodes execute in restricted sandbox.
- No env access from generated nodes (lint enforced).
- OPA/Rego or equivalent policy checks (production-grade subset).
- Signed artefact provenance — every artefact's hash anchored in `audit_external_anchor` (reusing S7).
- Isolated preview project with no production credentials.
- **Human approval mandatory** before G0 → G1 (out of scope this sprint, but the gate is wired).

### Out of Scope

- Auto-promotion beyond G0 — every G0→G1 needs human review.
- Real-data generated flows — Builder only handles synthetic.
- Marketplace/connectors ecosystem — bring-your-own connector deferred.
- Voice input — text only this sprint.
- AI-generated custom cockpit UI — flows + datasets only.

## Technical Implementation

### Deterministic FSM

`gateway/src/gateway/builder/session.py`:

```python
class BuilderState(StrEnum):
    INTAKE = auto(); RESEARCH = auto(); PLAN = auto()
    APPROVE = auto(); BUILD = auto(); LINT = auto()
    PREVIEW = auto(); DEPLOY = auto(); FAILED = auto()

ALLOWED = {
    BuilderState.INTAKE:   {BuilderState.RESEARCH, BuilderState.FAILED},
    BuilderState.RESEARCH: {BuilderState.PLAN, BuilderState.FAILED},
    BuilderState.PLAN:     {BuilderState.APPROVE, BuilderState.RESEARCH, BuilderState.FAILED},
    BuilderState.APPROVE:  {BuilderState.BUILD, BuilderState.PLAN, BuilderState.FAILED},
    BuilderState.BUILD:    {BuilderState.LINT, BuilderState.FAILED},
    BuilderState.LINT:     {BuilderState.PREVIEW, BuilderState.BUILD, BuilderState.FAILED},
    BuilderState.PREVIEW:  {BuilderState.DEPLOY, BuilderState.FAILED},
    BuilderState.DEPLOY:   set(),
    BuilderState.FAILED:   set(),
}
```

PG advisory lock on `(tenant_id, session_id)` ensures concurrent operator turns serialise.

### Bundle lint (OPA-style policy)

```rego
package builder.lint
default allow = false

allow {
  manifest_present
  egress_in_allowlist
  not env_access_in_code_nodes
  not literal_secrets
}
```

Lint runs with the AXA-curated allowlist (insurance APIs, internal MCP routers). Anything else fails. Failures returned as structured findings the UI can render.

### Citation discipline

`web_search_insurance` returns only allowlisted domains (regulators, AXA Group docs). The plan generator MUST cite for any factual claim or include "research pending — needs human review" in that section.

### Isolated preview project

Preview runs against a dedicated Railway environment `gdai-cockpit-preview` with:
- Separate Supabase project (no production rows reachable).
- Separate Langfuse project.
- Egress restricted to MCP-tool stubs only.
- Time-boxed (max 5 min per preview).

### Signed artefact provenance

Every generated bundle includes a `provenance.json`:
```json
{
  "builder_session_id": "...",
  "operator": "...",
  "plan_sha": "...",
  "build_sha": "...",
  "lint_status": "pass",
  "preview_run_id": "...",
  "signed_by": "project-pubkey-id",
  "signature": "..."
}
```
Hash anchored in `audit_external_anchor` from S7.

## Testing Plan

**Unit:**
- FSM transition table denies disallowed jumps.
- Lint rules: each guardrail individually triggers a failing example.
- Validators: anti-PII catches embedded `john@example.com`, IBAN, phone.
- Citation tool: non-allowlisted domain rejected.

**Integration:**
- Pre-S8 gate: 5 diverse briefs run end-to-end. Pass criteria measured.
- Concurrent operator turns serialise correctly via advisory lock.
- Preview runs cannot reach production Supabase (proven by deliberate misconfig + reject).

**Contract:**
- `POST /builder/sessions` → session id + initial intake question.
- `POST /builder/sessions/{id}/turn` → next state, plan or build artefacts.
- `POST /builder/sessions/{id}/approve` → only valid in `PLAN` state.

**Failure tests:**
- Prompt tries to exfiltrate env vars → lint blocks; finding reported.
- Generated HTTP node targets external URL not in allowlist → lint blocks.
- Citation search returns no hits → plan does not hallucinate; explicit "research pending".
- NIM unavailable → graceful pause/fallback, no partial deployment.
- Concurrent operator turns serialise correctly (advisory lock test).
- Preview run attempts a real side-effect (write production Postgres) → gateway dry-runs/blocks + audit row.
- Generated artefact missing capability manifest → deploy blocked.

## Sprint Review / Decision Gate

### Demo Script (20 min)

1. **(persona: builder operator / admin)** Open `/builder`. Type a brief: "A pilot for residential burst-pipe claims in France: triage, assess severity, pre-author letter to homeowner, dispatch plumber from approved list."
2. Watch the Builder do **research** with citations from allowlisted sources. Plan v0.1 appears with sections.
3. Operator revises: "Add a step for fraud risk score below threshold X."
4. Operator approves. Watch **build** — Monaco diff shows generated flow, dataset, eval rubric. Flow graph preview renders.
5. **Lint** runs — green.
6. **Preview run** in the isolated environment — full trace appears in the preview Langfuse project. Operator inspects steps.
7. Show negative tests:
   - Brief asks the bundle to exfiltrate `process.env.SUPABASE_SERVICE_ROLE_KEY` → lint blocks with specific finding.
   - Brief tries to add an HTTP node to `https://attacker.example/exfil` → blocked.
   - Citation search returns no hits → plan section says "research pending — human review needed", does not invent citations.
8. **Deploy G0** → new pilot card appears on `/pilots` with `gate=G0` badge. Bundle hash anchored in `audit_external_anchor`.
9. Open the new pilot, run a synthetic claim through it, watch live trace.
10. **Decision ask:** Is 4/5 reliability enough for an internal builder? Should security approval be a separate role from business approval? Skills-pack prompting vs dynamic skill routing? Generated pilots discovery-only or eventual production hardening?
